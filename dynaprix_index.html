Project Overview
WHAT IT
SOLVES

Traditional retail pricing relies on static margins and manual adjustments — missing revenue daily as market conditions shift. DynaPriX replaces guesswork with a data-driven recommendation engine that predicts optimal selling prices every purchase cycle.

By integrating internal ERP data with external macroeconomic signals — inflation, weather, fuel costs, and holiday calendars — the system captures demand fluctuations that static models structurally miss.

Pipeline Deep-Dive
📥
Stage 01
Data Ingestion
Raw data is pulled daily from the ERP system (purchase orders, sales records, inventory snapshots) and merged with external APIs: weather station data, INCC inflation scraping, and ANP diesel price feeds. A consolidated daily DataFrame spans January 2024 – April 2025 (~16 months).
ERP Integration Weather API INCC Scraper ANP Diesel Feed Daily Cadence
🧹
Stage 02
Data Cleaning
Each data source is inspected independently for missing values and outliers. Outlier treatment is context-driven (e.g. holiday spikes preserved, data-entry errors removed). Column types are standardized and date formats normalized across all joined sources.
Outlier Treatment Null Handling Type Normalization EDA
⚗️
Stage 03
Feature Engineering
The feature set is crafted to capture both time-series dynamics and nonlinear market effects. Demand lags capture recency signals. Rolling volatility measures price instability. Numerical features are quantile-binned to expose threshold effects that tree models can exploit.
Price Elasticity Formula
η = (ΔQ/Q) / (ΔP/P)  =  d(ln Q) / d(ln P)
Demand Prediction (simplified)
D(p) = D_base × exp(−η × (p − p_ref) / p_ref) × S(context)
Lag Features (7d/14d) Rolling Volatility Inventory Turnover Holiday Bins Macro Quantiles
🤖
Stage 04
Dual-Model Training
Two gradient-boosted tree models are trained independently on the feature set. LightGBM uses leaf-wise growth for efficiency on large feature sets; XGBoost uses level-wise growth as a regularized counterpart. Optuna runs 200 trials of Bayesian hyperparameter optimization for each model. Final evaluation uses MAE, RMSE, and WAPE on a held-out test set.
LightGBM v2 XGBoost v2 Optuna Tuning MAE · RMSE · WAPE Train/Test Split
💰
Stage 05
Profit Simulation
Given a purchase event, context features are auto-generated. A price sweep from ₹25 to ₹5000 is run through the ensemble model, predicting demand at each price point. Total profit and per-unit profit are computed. Business constraints (min margin, max deviation) are applied before the optimal price is surfaced.
Profit Optimization
P*(p) = argmax[(p − c) × D̂(p)]  s.t. p ≥ c × (1+min_margin)
Price Sweep Demand Prediction Margin Constraints Optimal ₹ Output
🚀
Stage 06
Recommendation & Deployment
The optimal price is returned to the ERP system as a pricing recommendation event. The system is designed for A/B testing deployment — 50% of SKUs receive model prices, the rest baseline. Monthly drift monitoring tracks residual patterns to trigger model retraining.
ERP Integration A/B Testing Ready Drift Monitoring Auto Retraining
